### 1. Error "Error al subir imagen a imgbb" — Voucher upload

**Problema:**  
El cliente subía el voucher y aparecía "Error al subir. Intenta de nuevo." sin más detalle.  
Luego con fix de sesión anterior se empezó a mostrar "Error al subir imagen a imgbb".

**Causa raíz:**  
En Railway, la variable `IMGBB_API_KEY` fue ingresada con comillas literales en el valor:
```
"36eaed76952b26c5c35263e22ae8597c"   ← incorrecto (con comillas)
36eaed76952b26c5c35263e22ae8597c    ← correcto (sin comillas)
```
Railway guardó las comillas como parte del valor, entonces imgbb recibía una key inválida y rechazaba el upload.

**Fix Railway:**  
Editar `IMGBB_API_KEY` → borrar las comillas del valor → guardar.

**Fix backend** (`backend/app/routers/pagos.py`):  
Mejorado el manejo de error en `subir_imagen_imgbb()` para mostrar el mensaje real que devuelve imgbb en vez de un error genérico:

```python
if response.status_code != 200:
    try:
        err = response.json()
        msg = err.get("error", {}).get("message", f"HTTP {response.status_code}")
    except Exception:
        msg = f"HTTP {response.status_code}"
    raise HTTPException(status_code=500, detail=f"Error imgbb: {msg}")
data = response.json()
if not data.get("success"):
    raise HTTPException(status_code=500, detail=f"imgbb no exitoso: {data}")
return data["data"]["url"]
```

### 2. Build APK — "Failed to remove build"

**Problema:**  
`flutter clean` muestra: *"Failed to remove build. A program may still be using a file in the directory"*

**Causa:**  
Algún proceso (IDE, emulador) tiene bloqueada la carpeta `build`.

**Fix:**  
Cerrar Android Studio / VS Code y ejecutar directamente:
```powershell
flutter pub get
flutter build apk --release
```
El `flutter clean` ya borró `.dart_tool` y `.flutter-plugins-dependencies` — suficiente para un build limpio.
