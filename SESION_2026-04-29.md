# Sesión 29-04-2026 — Resumen de cambios

## Fix 1 — Error 422 al subir voucher (rol cliente)

**Problema:** Al subir la imagen del Yape en `pago_modal.dart`, el API devolvía 422.

**Causa:** `Options(contentType: 'multipart/form-data')` en Dio sobreescribe el boundary
autogenerado del FormData, haciendo que FastAPI no pueda parsear el body.

**Fix:** Quitar el parámetro `options` del `.post()` — Dio genera el boundary correcto solo.

```dart
// ANTES (roto)
await ApiClient().dio.post('/pagos/${widget.pagoId}/voucher',
    data: formData,
    options: Options(contentType: 'multipart/form-data'));

// DESPUÉS (correcto)
await ApiClient().dio.post('/pagos/${widget.pagoId}/voucher', data: formData);
```

**Archivo:** `frontend/lib/shared/modals/pago_modal.dart`

## Fix 5 — Precios día/noche no se mostraban a los clientes (fix definitivo)

**Problema:** Los precios `precio_dia` y `precio_noche` configurados por el admin
no aparecían en la vista del cliente. La lógica dependía de `HorarioDisponible`,
que puede no existir o tener `activo=False`.

**Causa raíz:** Los precios solo se guardaban en `HorarioDisponible.precio_override`,
no en el modelo `Cancha` directamente.

**Fix definitivo — columnas nuevas en `canchas`:**


**Migración:** `backend/migrations/versions/f7a8b9c0d1e2_add_precio_dia_noche_to_canchas.py`

**Archivos actualizados:**

## Fix 7 — Formato de fechas unificado a dd-MM-yyyy en todos los roles

**Problema:** Las fechas se mostraban en distintos formatos según el rol y la pantalla
(EEEE d MMM yyyy, dd/MM, yyyy-MM-dd crudo, etc.).

**Regla:** Toda fecha visible al usuario debe mostrarse como `dd-MM-yyyy` (ej. 29-04-2026).
Las fechas enviadas al API se mantienen en `yyyy-MM-dd`.

**Helper agregado en cada State:**
```dart
String _fmt(String? f) {
  if (f == null || f.isEmpty) return '—';
  final dateStr = f.contains('T') ? f.split('T')[0] : f; // soporta ISO datetime
  final p = dateStr.split('-');
  if (p.length == 3) return '${p[2]}-${p[1]}-${p[0]}';
  return f;
}
```

## Error APK — `_fmt` en clase equivocada

**Error:** `Target kernel_snapshot_program failed: Exception`
```
_fmt isn't defined for the class '_DetalleReservaSheet'
_fmt isn't defined for the class '_AdminBloqueosPageState'
```

**Causa:** `_fmt` se agregó a la clase principal (`_State`), pero en esos dos archivos
hay clases secundarias separadas que también usan `_fmt`:
- `admin_bloqueos_page.dart`: `_fmt` quedó en `_FormularioBloqueoState`, se necesitaba en `_AdminBloqueosPageState`
- `admin_reservas_page.dart`: `_fmt` quedó en `_State`, se necesitaba en `_DetalleReservaSheet` (StatelessWidget)

**Fix:** Se agregó `_fmt` también en la clase que la usa en cada archivo.

**Lección:** Cuando un archivo tiene múltiples clases (sheets, dialogs como StatelessWidget
o StatefulWidget separados), agregar el helper en CADA clase que lo use, o definirlo
como función top-level (fuera de toda clase) para que sea accesible desde todo el archivo.

## Consideraciones generales para próximas sesiones

- **Multipart en Dio:** Nunca pasar `Options(contentType: 'multipart/form-data')` —
  Dio genera el boundary correcto automáticamente con `FormData`.

- **Nav bar overlap:** Siempre usar `MediaQuery.of(context).padding.bottom` en modales/sheets
  que tienen botones al fondo. Sin esto el botón queda tapado en celulares con gesture nav.

- **Precios en Cancha:** `precio_hora`, `precio_dia`, `precio_noche` se leen/escriben
  directo del modelo `Cancha`, NO de `HorarioDisponible.precio_override`.

- **Fechas:** API recibe/devuelve `yyyy-MM-dd`. UI muestra `dd-MM-yyyy`. Usar `_fmt()` para convertir.

- **Notificaciones push:** Ya están configuradas. El super admin recibe push cuando
  llega un voucher de suscripción. No duplicar lógica.

- **Railway:** Corre Alembic en cada deploy automáticamente. Las migraciones nuevas
  se aplican solas al hacer push.

- **APK build:** Copiar carpeta PICHANGAYA a `E:\PICHANGAYA`, abrir PowerShell en
  `E:\PICHANGAYA\frontend` y correr `flutter build apk --release`.
  El APK queda en `build\app\outputs\flutter-apk\app-release.apk`.
